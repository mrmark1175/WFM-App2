# WFM Prototype: Project Capabilities Audit
**Date:** March 16, 2026
**Status:** Functional Prototype / Alpha

## 1. Executive Summary
The WFM-App2 is a sophisticated Workforce Management (WFM) platform built with a modern React/Vite/Node.js stack. It transitions traditional spreadsheet-based forecasting and capacity planning into a centralized, database-driven web application. The system is designed for high-precision staffing calculations using Erlang-C and advanced time-series forecasting.

---

## 2. Core Functional Modules

### A. Advanced Forecasting (`Forecasting.tsx`)
*   **Statistical Engine:** Employs **Holt-Winters (Triple Exponential Smoothing)** to project future call volumes.
*   **Seasonality Handling:** Configurable Alpha (level), Beta (trend), and Gamma (seasonality) parameters.
*   **Historical Integration:** Automatically detects years and months from interval data in the PostgreSQL database.
*   **Visualization:** Interactive Recharts-based dashboards comparing "Actuals" vs. "Projections."

### B. Strategic Capacity Planning (`CapacityPlanning.tsx`)
*   **Erlang-C Engine:** Calculates required staffing (FTE) based on Service Level (SL) targets, Average Handle Time (AHT), and Abandonment goals.
*   **Workforce Modeling:** 
    *   **Attrition Tracking:** Models monthly/weekly FTE decay.
    *   **Training Pipeline:** Supports adding training classes with configurable "Ramp-up" periods.
*   **Scenario Comparison:** Allows users to save multiple "What-If" scenarios to the database for long-term budget and hiring planning.

### C. Arrival & Trend Analysis (`ArrivalAnalysis.tsx`)
*   **Granular Drilldowns:** View data across multiple hierarchies:
    *   **YoY:** Long-term growth trends.
    *   **Monthly/Weekly:** Seasonal peaks.
    *   **Intraday:** 15/30/60-minute interval profiles.
*   **Heatmap Visualization:** Visual identification of "Hot" intervals for both Volume and AHT.
*   **Distribution Analysis:** Calculates interval-of-day percentages, essential for schedule generation.

### D. Data Ingestion (`InteractionArrival.tsx`)
*   **Telephony Integration:** Structural support for **Genesys Cloud**, Avaya, and Five9.
*   **API Middleware:** A Node.js backend handles high-speed batch inserts and "Upserts" for interval data.
*   **Manual Overrides:** Capability to adjust interval data directly within the grid.

## 3. Technical Stack Audit

### Frontend (React/TypeScript)
*   **UI Components:** Radix UI / Shadcn (for high-fidelity accessible components).
*   **State Management:** React Hooks (UseState/UseEffect/UseMemo) for complex data grids.
*   **Styling:** Tailwind CSS (Modern utility-first styling).

### Backend (Node.js/Express)
*   **Database:** PostgreSQL (`exordium_db`) for robust relational data storage.
*   **API Layers:** RESTful endpoints for agent rosters, forecasts, and capacity scenarios.
*   **Integrations:** `purecloud-platform-client-v2` included for Genesys Cloud connectivity.

## 4. Current Limitations & Roadmap Recommendations

1.  **Intraday Real-time:** The `IntradayForecast.tsx` module is currently a visual placeholder and requires real-time data stream integration.
2.  **Authentication:** The app currently operates without a login layer (Identity Management).
3.  **Scheduling:** While the app calculates *requirements*, it does not yet generate individual agent schedules (Roster-to-Interval mapping).
4.  **Genesys Credentials:** Integration is coded but requires environment-specific Client IDs and Secrets to be populated in `server/genesys.cjs`.
